# Unified-Policy Run

This folder consolidates the four main figures of the manuscript under a **single, unified policy definition**:
all four figures share the same `reference_policy` definition from
[`waste_trade_policy_constraints_reference_relaxed.csv`](data/fig2/waste_trade_policy_constraints_reference_relaxed.csv).

## Why this folder exists

Earlier the four figures used two different `reference_policy` definitions:

- **Standard policy** (`waste_trade_policy_constraints_with_critical_routes.csv`): reference forbids
  AnnexVII -> NonAnnexVII shipments (Basel Ban Amendment baseline).
- **Relaxed policy** (`waste_trade_policy_constraints_reference_relaxed.csv`): reference allows
  AnnexVII -> NonAnnexVII shipments with $50/t friction (Basel Ban relaxation sensitivity).

Fig 2 was generated under the relaxed policy; Fig 3 and Fig 4 used the standard policy. To eliminate
the cross-figure inconsistency we re-ran Fig 3 and Fig 4 under the relaxed policy and copied all four
figures together with their input data into this folder.

## Reproducibility

1. **Fig 1** (D-panel cascade): does not depend on `reference_policy`; copied from existing project outputs.
2. **Fig 2** (route maps): produced earlier under reference_relaxed; data and PNGs copied here verbatim.
3. **Fig 3** (PyroHydro robustness, S1-S5): regenerated by

   ```bash
   python code/run_dynamic_scale_policy_technology.py \
     --years 2025,2026,...,2050 \
     --include-pyrohydro \
     --pyrohydro-pyro-weight <S-specific> \
     --pyrohydro-group-multipliers <S-specific> \
     --lp-solver gurobi --skip-fixed \
     --output-dir data/fig3_pyrohydro_sensitivity_unified/<scenario_name>
   ```
4. **Fig 4** (mitigation marginal benefit): regenerated by

   ```bash
   python code/run_lithium_loss_scenarios.py --include-max-li
   ```
